Ma'lumotlar saqlanishi, Admin tizimi ishlashi va AI integratsiyasi real bo‘lishi shart.1. Texnologik Stack (Qat'iy rioya qiling)Core: Python 3.11+ (Flask Framework).Database: SQLite (SQLAlchemy ORM). Restart berganda o'chib ketmasligi kerak.Frontend: Server-side Rendering (Jinja2) + TailwindCSS (CDN versiyasi). Node.js yoki npm build ishlatilmasin.AI: Google Gemini API (google-generativeai kutubxonasi).Auth: Flask-Login (User Session Management).Security: Werkzeug (Parol xeshlash), CSRF Protection.2. Fayl Strukturasi (Architectural Blueprint)Loyiha fayllarini quyidagi tartibda yarating:main.py: Ilovaning kirish nuqtasi, config va route-lar.extensions.py: db, login_manager initsializatsiyasi (Circular Import oldini olish uchun).models.py: Barcha DB modellari (User, Product, Order, OrderItem).services.py: AI logikasi va Rasmga ishlov berish funksiyalari.utils.py: Yordamchi funksiyalar (masalan, fayl nomidan narxni ajratish regexi).templates/: HTML fayllar (base.html, admin_dashboard.html, shop.html, product_detail.html, cart.html, login.html).static/css/: Maxsus stillar (agar Tailwind yetmasa).static/uploads/: Yuklangan rasmlar papkasi.3. Ma'lumotlar Bazasi (Schema)User: id, username, password_hash, is_admin (Boolean).Product: id, title, price (Integer), description (Text), image_url, tags (JSON), created_at.Order: id, customer_name, phone, total_amount, status ('new', 'processing', 'completed'), created_at.OrderItem: id, order_id, product_id, quantity, price_at_purchase.Muhim: Ilova har safar ishga tushganda bazani tekshirsin. Agar admin user yo'q bo'lsa, avtomatik yaratsin (Login: admin, Parol: admin123).4. Biznes Logikasi va AI Integratsiyasi (Core Logic)Scenario 1: Mahsulot Qo'shish (Admin)Admin /admin/upload sahifasida rasm yuklaydi (masalan: krasovka_700ming.jpg).Backend (Regex): Fayl nomidan narxni qidiradi (\d+(ming|mln|som)). Agar topmasa, default 0 qo'yadi.Backend (AI Service): Rasmni Gemini 2.5 Flash ga yuboradi.Prompt: "Bu mahsulot rasmi. Menga JSON formatda qaytar: title (qisqa nom), description (sotuvchi matn, 2 gap), tags (3 ta hashtag)."Natija olinib, ma'lumotlar bazasiga (Product jadvaliga) yoziladi.Scenario 2: Savdo va Savat (Client)Mijoz "Savatga qo'shish" tugmasini bosadi -> Ma'lumot session['cart'] da saqlanadi (DB shart emas)."Buyurtma berish" sahifasida Ism va Telefon kiritadi."Tasdiqlash" bosilganda -> Order va OrderItem jadvallariga yoziladi va Savat tozalanadi.Mijozga "Buyurtmangiz qabul qilindi ID: #123" degan chiroyli sahifa chiqadi.5.
